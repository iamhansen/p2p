Introduction To BitTorrent

Summary

BitTorrent is a Protocol using to share files with other computers across the internet.
It was originally developed by Bram Cohen, and released in July 2001.

Usage

Using a BitTorrent client (or the original one) originally comprised of
  • A Client
  • A .torrent file
  • A Tracker Server
However, in newer clients, it is able to use MAGNET links as a replacement for .torrent files,
and Distrubuted Hash Tables, Local Peer Discovery, and Peer Exchange. (DHT, LPD, PEX)

Using BitTorrent you join a 'swarm' of computers that download/upload to each other.
To share a file, the person sharing at first must make a torrent file, which can be made by their BitTorrent client.
The torrent file consists of metadata to find and reassemble the file.

The metadata is the title of the torrent, trackers to use, list of files, comments, piece size & hashes, info hash, and miscellaneous info. (comments, other torrents, source)
When sharing files with BitTorrent, the client is uploading/downloading segments of the file called pieces.
The file is segmented into equal sized (usually from 32kb to 16mb) pieces.
Due to this segmentation, any torrent download can be paused and restarted without much data loss.
Each piece has their own hash to be stored in the torrent file with all the other metadata.
This is to easily check for any modifications of the file to prevent accidental or malicious data from being sent to others.
